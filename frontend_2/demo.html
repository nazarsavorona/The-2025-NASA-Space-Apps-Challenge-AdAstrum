🌌 Exoplanet Visualizer

Interactive 3D visualization of exoplanets and star systems using real NASA data

✨ Features

🪐 3D Visualization

Stunning WebGL-based 3D rendering of exoplanets with realistic textures

🌟 Star Systems

View complete planetary systems with orbital mechanics

📊 Real Data

All data from confirmed NASA exoplanet discoveries

🎮 Interactive

Control camera, animation, and switch between view modes

🚀 Quick Start

⚠️ Prerequisites: Make sure you have Node.js (v16+) and Python 3.8+ installed

Option 1: Automated Start (Recommended)

$ cd frontend_2
$ ./start.sh

Option 2: Manual Start

Step 1: Install frontend dependencies

$ cd frontend_2
$ npm install

Step 2: Start backend server (in a new terminal)

$ cd backend
$ pip install -r requirements.txt
$ python main.py

Step 3: Start frontend (in another terminal)

$ cd frontend_2
$ npm run dev

Step 4: Open your browser to http://localhost:3000

🪐 Available Exoplanets

  • Kepler-186 f - First Earth-size planet in the habitable zone (492 light-years away)
  • Kepler-22 b - Ocean candidate planet 2.4x Earth's radius (620 light-years)
  • TRAPPIST-1 e - Rocky temperate world in 7-planet system (39.6 light-years)
  • HD 209458 b - Famous hot Jupiter with evaporating atmosphere (159 light-years)
  • GJ 1214 b - Water-rich sub-Neptune around red dwarf (48 light-years)

🛠️ Technology Stack

Spacekit.js - 3D Space Visualization WebGL - Hardware-accelerated Graphics Vite - Fast Build Tool FastAPI - Python Backend Vanilla JavaScript - Pure Performance

🎮 Controls

  • Select Planet: Choose from the dropdown menu
  • View Mode: Switch between single planet and star system view
  • Mouse Drag: Rotate the camera around the scene
  • Mouse Wheel: Zoom in and out
  • Play/Pause: Control the animation
  • Reset: Return camera to default position

🔧 Customization

Adding new planets: Edit backend/data/planets.json

Changing colors: Modify src/style.css

Adjusting physics: Edit simulation parameters in src/main.js

🐛 Troubleshooting

Port already in use? Kill the process or change the port in configuration files

Dependencies not installing? Try deleting node_modules and package-lock.json, then reinstall

Textures not loading? Check that the backend is serving files from the correct directory
